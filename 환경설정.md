## 클라우드 실습 프로젝트

### 프로젝트의 기본 디렉토리
1)bin 
- www 이름의 파일이 한개 존재. 서버를 시작할 포트가 지정됨. 
2)node_modules
- npm i 실행하면 생김. package.json에 선언된 모듈과 이 모듈을 <br>
실행하기 위해 필요한 의존관계 모듈이 설치됨.
3)public
- 이미지, 자바스크립트, 스타일 시트 같은 정적 리소스 관리함.
4)routes
- index.js와 users.js 파일이 존재함.
- 디폴트로 생성되는 파일이다
- 라우트를 처리하기 위한 코드가 있다
5)views
- 뷰를 처리하는 파일이 위치함.
- 프로젝트 생성 시 템플릿 엔진으로 ejs를 지정함.
- 이 디렉토리에는 확장자가 ejs인 파일이 위치함.

### 디렉토리 복습
cloud-app
/bin/www - 포트번호를 변경가능함.

/public/images - 이미지 리소스 저장
/public/javascripts/firebase.js - 파이어베이스API활용

/routes/index.js - 현재는 여기에 엔드포인트 추가(GET, POST-파일업로드, 프사) 
/routes/users.js - 인증 후 인가된 요청URL(예: 장바구니 엔드포인트)
- https://cloudinary.com/ - 클라우드 서비스를 이용한 이미지 업로드 처리(모듈화 실습)
- 모듈화(CJS&ESM) 
- Front-End(View계층, UI-UserInterface)
/views/auth
/views/common - 모달
/views/include - jsp, ejs, header.ejs, footer.ejs
/views/pages/
